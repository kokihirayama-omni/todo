# タスク09: メインページの実装

## 概要
タスク一覧表示と操作を行うメインページを実装します。

## ファイル
`app/page.tsx`

## 実装内容

### 1. クライアントコンポーネント化
```typescript
"use client";
```

### 2. 必要なインポート
- `useTaskStore` - タスク状態管理
- `useNotification` - 通知フック
- `TaskList` - タスクリストコンポーネント
- `TaskModal` - タスク追加/編集モーダル
- その他のUIコンポーネント

### 3. ページ構成
```
+--------------------------------------------------+
| Header (ヘッダー)                                 |
+--------------------------------------------------+
| [フィルター] [ソート順]                           |
|                                                  |
| TaskList (タスク一覧)                             |
|   - TaskCard                                     |
|   - TaskCard                                     |
|   - TaskCard                                     |
|                                                  |
| [+ 新規タスク追加]                                |
+--------------------------------------------------+
```

### 4. 機能実装
- 通知フックの初期化（`useNotification()`）
- 擬似認証フローの実装（初回起動時のダイアログ）
- タスクの一覧表示
- フィルタリング機能
  - すべて
  - 未完了のみ
  - 完了済みのみ
- ソート機能
  - 期限日時順
  - 作成日時順
  - 優先度順

### 5. 新規タスク追加
- ボタンクリックで TaskModal を開く
- モーダル内でタスク情報を入力
- 送信時に `addTask` を呼び出す

## レイアウト
- 中央寄せのコンテナ（max-width: 1200px程度）
- 適切なパディング・マージン
- レスポンシブ対応

## 状態管理
- ローカルステート
  - `showModal: boolean` - モーダルの表示状態
  - `editingTask: Task | null` - 編集中のタスク
  - `filter: 'all' | 'active' | 'completed'` - フィルター状態
  - `sortBy: 'dueDate' | 'createdAt' | 'priority'` - ソート順

## 注意事項
- LocalStorageからの読み込みは `useEffect` 内で行う
- Next.jsのApp Routerでは、クライアントコンポーネントとして実装
