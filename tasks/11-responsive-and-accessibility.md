# タスク11: レスポンシブデザインの調整とアクセシビリティの確認

## 概要
モバイル/デスクトップ両対応のレスポンシブデザインを調整し、アクセシビリティを確保します。

## レスポンシブデザイン

### ブレークポイント（Tailwindデフォルト）
- `sm`: 640px
- `md`: 768px
- `lg`: 1024px
- `xl`: 1280px

### モバイル（〜768px）
- シングルカラムレイアウト
- ヘッダーの要素を縦積み
- タッチ操作に適したボタンサイズ（最小44x44px）
- フォントサイズの調整

### タブレット（768px〜1024px）
- 2カラムグリッド
- ヘッダーは横並び

### デスクトップ（1024px〜）
- 3カラムグリッド
- 最大幅を制限（max-w-7xl等）

### 調整が必要な箇所
1. **Header**
   - モバイル: タイトルとボタンを縦積み
   - デスクトップ: 横並び
2. **TaskList**
   - モバイル: 1カラム
   - タブレット: 2カラム
   - デスクトップ: 3カラム
3. **TaskModal**
   - モバイル: 全画面表示
   - デスクトップ: 中央に固定幅
4. **フォーム要素**
   - タッチデバイスでの操作性を考慮

## アクセシビリティ（a11y）

### キーボード操作
- [ ] すべてのインタラクティブ要素にフォーカス可能
- [ ] Tab/Shift+Tab でのナビゲーション
- [ ] Enter/Space でのボタン操作
- [ ] Escape でモーダルを閉じる

### セマンティックHTML
- [ ] 適切な見出しレベル（h1, h2, h3）
- [ ] `<button>` と `<a>` の使い分け
- [ ] フォームラベルの関連付け

### ARIA属性
- [ ] `aria-label` で説明が必要な要素に付与
- [ ] `aria-describedby` でエラーメッセージを関連付け
- [ ] `role` 属性（必要に応じて）
- [ ] `aria-live` で動的な変更を通知

### カラーコントラスト
- [ ] WCAG AA基準を満たす（最低4.5:1）
- [ ] 優先度バッジの色のコントラストチェック

### フォーカススタイル
- [ ] フォーカス時に視覚的なインジケーター表示
- [ ] `focus-visible` の活用

### スクリーンリーダー対応
- [ ] 画像に `alt` 属性
- [ ] アイコンのみのボタンに `aria-label`
- [ ] 状態変化の適切な通知

## テスト項目

### レスポンシブ確認
- [ ] Chrome DevToolsで各ブレークポイントを確認
- [ ] 実機（iPhone, Android, iPad）での確認

### アクセシビリティ確認
- [ ] キーボードのみで全操作が可能か
- [ ] Lighthouse（Accessibility）スコア90点以上
- [ ] axe DevToolsでエラーがないか
- [ ] スクリーンリーダー（VoiceOver/NVDA）での動作確認

## shadcn/uiの利点
shadcn/uiはRadix UIベースのため、以下がデフォルトで対応済み：
- キーボードナビゲーション
- ARIA属性
- フォーカス管理

## 修正が必要になった場合
- Tailwind CSSのユーティリティクラスで調整
- カスタムメディアクエリが必要な場合は `@media` を使用
